# ROADWATCH API Documentation

### Base Setup
* Base URL: http://localhost:8000 (or your deployed server domain)
* CORS: Cross-Origin Resource Sharing is enabled globally for all routes.
* Content-Type: For POST requests, the frontend must set the header Content-Type: application/json.

---

## Authentication Endpoints

### 1. Initiate Google Login
* Endpoint: GET /login
* Description: Redirects the user to Google's OAuth consent screen. The frontend can use this endpoint as the href for a "Login with Google" button.
* Response: A redirect to accounts.google.com.

### 2. OAuth Callback
* Endpoint: GET /authorize
* Description: This is the callback URI used by Google. Upon successful authentication, it registers/checks the user in the database and returns their Google account details.
* Response (200 OK):
  {
    "email": "user@example.com",
    "name": "User Name",
    "openid": "1234567890..." 
  }
* Response (400 Bad Request):
  {
    "error": "Failed to fetch user info"
  }

### 3. Get User Profile
* Endpoint: GET /user/<openid>
* Description: Fetches user data from the database using their Google OpenID.
* Path Parameter:
  * openid (string): The unique Google user ID returned from the /authorize endpoint.
* Response: Returns a JSON object containing the user's data (structure depends on db.check_user).

## Data Retrieval Endpoints

### 4. Get States
* Endpoint: GET /states
* Description: Retrieves a list of all supported states from the system.
* Response (200 OK):
  {
    "states": ["WB", "MH", "KA", "..."]
  }

### 5. Get Vehicle Types
* Endpoint: GET /vehicle-types
* Description: Retrieves a list of all supported vehicle types (e.g., Bike, Car, Truck).
* Response (200 OK):
  {
    "vehicle_types": ["Bike", "Car", "Truck", "..."]
  }

### 6. Get Violations
* Endpoint: GET /violations
* Description: Retrieves a list of all trackable traffic violations.
* Response (200 OK):
  {
    "violations": ["No Helmet", "Over Speeding", "..."]
  }

## Functional Endpoints

### 7. Calculate Fine
* Endpoint: POST /calculate-fine
* Description: Calculates the fine amount based on the state, vehicle type, specific violation, and whether it's a repeat offence. Note: Inputs are processed case-insensitively on the backend.
* Request Body (JSON):
  {
    "state": "WB",
    "vehicle_type": "Bike",
    "violation": "No Helmet",
    "first_offence": false
  }
  (Note: first_offence must be a boolean: true or false)

* Response (200 OK):
  {
    "fine": 1000
  }
* Response (404 Not Found):
  {
    "error": "No matching challan rule found"
  }
* Response (400 Bad Request):
  {
    "error": "Missing field: <field_name>"
  }

### 8. Ask LLM (Chatbot)
* Endpoint: POST /ask-llm
* Description: Submits a query to the AI chatbot, along with relevant context like state or violation type.
* Request Body (JSON):
  {
    "user_query": "How much is the fine for drunk driving?",
    "state_code": "WB",
    "violation_id": "Drunk Driving", 
    "vehicle_type": "Car" 
  }
  (Note: user_query and state_code are required. violation_id and vehicle_type are optional).

* Response (200 OK):
  Returns a dynamic JSON response generated by the LLM chatbot (depends on the AI's internal implementation).
* Response (400 Bad Request):
  {
    "error": "user_query and state_code are required"
  }
* Response (500 Internal Server Error):
  {
    "error": "<exception_message_from_backend>"
  }
